# Changelog

Track important changes, releases, and security updates.

## Template

Use [`CHANGELOG_TEMPLATE.md`](CHANGELOG_TEMPLATE.md) for each milestone entry.

## When to update

- Major phase completion (Sources Phase 3+, OAuth, etc.)
- Breaking API or schema changes
- Security fixes
- Tagged releases

## Main changelog file

Create `CHANGELOG.md` at repo root when the first version tag ships. Until then, milestone entries may live in this folder as `YYYY-MM-DD-milestone.md`.
