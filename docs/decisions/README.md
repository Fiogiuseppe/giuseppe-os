# Decisions

Architecture Decision Records (ADRs) for Giuseppe OS.

## What belongs here

- Store backend choices
- Auth and token architecture
- AI provider strategy
- Data model boundaries (memory vs identity vs evidence)
- Integration patterns that are hard to reverse

## What does not belong here

- Routine bug fixes
- UI tweaks
- Phase reports (see [`../reports/`](../reports/))

## How to add a decision

1. Copy [`ADR-000-template.md`](ADR-000-template.md) → `ADR-NNN-short-slug.md`
2. Fill all sections; set status to **Accepted** when merged
3. Link from phase report and architecture doc
4. If superseded, update status — do not delete history
